# Emails Resend i18n FR/EN/DE — Rapport (Fix #156/#157)

## Récap

Tous les emails transactionnels passent désormais par un dictionnaire i18n
(`EmailLang = 'fr' | 'en' | 'de'`) et sont envoyés dans la langue du
destinataire (`users/{uid}.language`). Backward compatible : `renderTemplate()`
et `sendEmail()` sans paramètre `lang` continuent à produire du FR.

## Wrapper sendEmail

`src/lib/email/sendEmail.ts` — ajout `lang?: EmailLang` dans `SendEmailOptions`,
propagé à `renderTemplate(templateName, data, lang ?? 'fr')`.

## Helper getUserLang / pickUserLang

`src/lib/i18n/getUserLang.ts` :
- `getUserLang(uid, defaultLang='fr')` — async, lit `users/{uid}.language` via
  firebase-admin (pattern lazy init cohérent avec `notifyUser.ts`).
- `pickUserLang(userDocData)` — sync, ajouté pour les callsites qui ont déjà
  lu le snapshot user (évite un round-trip Firestore).

## Conformité doctrine

- Charte stricte préservée : `#D91CD2`, `#000000`, layout table-based, pas de
  gradient, inline styles email-safe.
- Variables `${d.xxx}` et balises `<strong style="...">` préservées dans les 3
  langues.
- Références juridiques traduites fidèlement :
  - FR : revFADP, nLPD, LCD
  - EN : revFADP, GDPR, UCA, FADP (kept Swiss-Romand naming)
  - DE : revDSG, DSGVO, UWG
- Noms de marque (Spordateur, Afroboost), emails (contact@spordateur.com) et
  références CGU (`section 7.bis`, `§F`, etc.) préservés.
- Backward compatible : tous les anciens appels sans `lang` produisent FR (tests
  E1-E4 + EM-INV1 + EM-SP1-4 + SR1-7 continuent de passer).
