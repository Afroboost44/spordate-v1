# Push FCM i18n — Rapport de refactor

## Objectif
Rendre les notifications push FCM traduisibles FR/EN/DE selon `users/{uid}.language`.

## Fichiers créés

### `src/lib/i18n/serverTranslations.ts`
Source de vérité serveur pour les templates push.

Exports :
- `type ServerLang = 'fr' | 'en' | 'de'`
- `const SUPPORTED_LANGS`, `const DEFAULT_LANG = 'fr'`
- `type MessageKey` : union des clés de message supportées
- `interface PushTemplate { title: string; body: string }`
- `const PUSH_TEMPLATES: Record<ServerLang, Record<MessageKey, PushTemplate>>`
- `function coerceLang(value: unknown): ServerLang` — coerce safe avec fallback
- `function tPush(lang, key, params?): { title, body }` — traduit + interpole `{key}`

### `src/lib/i18n/getUserLang.ts`
Helper `getUserLang(uid, defaultLang?)` qui lit `users/{uid}.language` via firebase-admin/firestore. Pattern lazy-init cohérent avec `notifyUser.ts`. Fallback `'fr'` si absent/invalide/erreur.

Note : `notifyUser` lui-même n'utilise PAS ce helper — il lit déjà le user doc pour `fcmToken`, donc il extrait `data_.language` au passage (évite un round-trip Firestore). `getUserLang` reste disponible pour les futurs callsites externes qui n'ont pas le user doc en main.

## Fichiers modifiés

### `src/lib/notifications/notifyUser.ts`
- Signature étendue (union discriminée) : `NotifyUserOptionsLegacy` (`title`/`body` bruts) | `NotifyUserOptionsI18n` (`messageKey` + `params?`).
- Backward-compat : les appelants legacy (`title`/`body`) fonctionnent sans changement.
- En mode i18n : la fonction lit `data_.language` du user doc déjà chargé (pas de read Firestore additionnel), coerce via `coerceLang`, et appelle `tPush(lang, messageKey, params)`.
- Import ajouté : `import { coerceLang, tPush, type MessageKey } from '@/lib/i18n/serverTranslations'`.

## Comment ajouter une nouvelle MessageKey

1. Étendre l'union `MessageKey` dans `src/lib/i18n/serverTranslations.ts`.
2. Ajouter l'entrée `{ title, body }` dans `PUSH_TEMPLATES.fr`, `.en`, `.de` (TypeScript force la complétude).
3. Au callsite : `notifyUser({ uid, messageKey: 'new_key', params: { ... } })`.
